Portfolio (Code)

Development
Individueel


Vanaf het begin dat ik hoorde dat we een portfolio bij moesten houden wist ik dat ik hier veel tijd in ging steken. Ik ben meteen aan de slag gegaan met een design waar ik feedback op had gevraagd van zowel medestudenten als een docent. Nadat ik een idee had hoe ik alles wilde gaan vormgeven ben ik begonnen met het programmeren van de site. Ik heb hiervoor HTML, CSS in de vorm van Sass, en JavaScript gebruikt. Het hele project heb ik bijgehouden op een openbare Github repository en dus kan de sourcecode bekeken worden.


Github

Structuur

Voor de structuur van mijn portfolio moest ik natuurlijk HTML gebruiken. Hier heb ik geprobeerd zo efficient en minimalistisch te werken zodat de code leesbaar bleef. Ook wilde ik de mappenstructuur goed in orde hebben voordat ik begon, dus heb ik deze eerst verzorgd.

Opmaak

Voor alle opmaak heb ik mijn eigen CSS geschreven in de vorm van Sass. Sass is een CSS-preprocessor waar ik al enige tijd gebruik van maak om mijn CSS efficiënter te schrijven. Denk hierbij aan variabelen en voorgeschreven CSS die hergebruikt kan worden op een eenvoudige manier. Als algemene grid-systeem maak ik gebruik van Bootstrap, echter gebruik ik hier alleen het grid van en niks anders qua CSS of JavaScript, omdat ik dit zelf wil schrijven.

Interactie

Ik wilde mijn portfolio een fijne UX geven door animaties te gebruiken die smooth zijn en niet afnemen van het eindproduct. Dit heb ik gedaan door een combinatie van CSS3 animaties te gebruiken en wat JavaScript. De JavaScript heb ik hier voornamelijk een library genaamd Animate On Scroll gebruikt om zo alle elementen op een mooie manier te laden. Voorheen heb ik al wel gebruik gemaakt van libraries die content op een mooie manier laden, echter heb ik nog niet veel met CSS3 animaties gedaan en hier heb ik veel in geleerd zoals de gekleurde cirkels op de homepagina en alle projecten die bij het laden van een bepaalde positie naar de correcte positie schuiven.

Wat heb ik geleerd?

Tijdens het maken van dit portfolio heb ik nieuwe dingen geleerd over het optimaliseren van JavaScript en te kijken welke libraries vervangbaar zijn door eigen code. Ook heb ik meer ervaring gekregen met de mogelijkheden van CSS3-transitions door gebruik te maken van css-transforms. Verder heb ik meer inzicht gekregen in het ontwerpen voor mobiele devices en rekening houden met responsiveness op gebied van CSS.

Wat kon beter?

Dingen die beter konden zijn vooral gebaseerd op feedback die ik heb gekregen. Om te zien welke feedback ik heb gekregen en hoe ik deze verwerkt heb is hier een case study over de UX van dit portfolio.
